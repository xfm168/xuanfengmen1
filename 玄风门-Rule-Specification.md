## 三、Rule 结构规范

### 3.1 标准结构

```typescript
interface BaseRule<TContext, TResult> {
  // 基本信息
  id: string              // 唯一ID，遵循编号规范
  name: string            // 规则名称（中文，便于阅读）
  category: string        // 分类
  subCategory?: string    // 子分类

  // 优先级与权重
  priority: number        // 优先级 0-999
  weight: number          // 权重 0-100

  // 描述与来源
  description: string     // 规则描述（一句话说明）
  detail?: string         // 详细说明
  reference: string       // 典籍来源
  referenceDetail?: string // 典籍具体章节/页码

  // 版本信息
  version: string         // 规则版本号
  createdAt: string       // 创建日期
  updatedAt: string       // 更新日期

  // 逻辑
  condition: (ctx: TContext) => boolean   // 条件函数
  result: TResult                         // 结果
}
```

### 3.2 必填字段

每一条 Rule 必须包含：

- [ ] `id` — 全局唯一，符合编号规范
- [ ] `name` — 中文名称，清晰易懂
- [ ] `category` — 分类
- [ ] `priority` — 优先级
- [ ] `weight` — 权重
- [ ] `description` — 一句话描述
- [ ] `reference` — 典籍来源
- [ ] `condition` — 条件函数
- [ ] `result` — 结果

### 3.3 可选字段

- `detail` — 详细解释
- `referenceDetail` — 典籍具体位置
- `examples` — 典型命例
- `conflictsWith` — 可能冲突的规则ID列表
- `dependsOn` — 依赖的前置规则

## 五、权重规范

### 5.1 权重定义

权重表示这条规则的**可信度/重要程度**，范围 0-100。

| 权重范围 | 含义 | 示例 |
|---------|------|------|
| 90-100 | 绝对可信 | 基础事实、典籍明确记载 |
| 70-89 | 高度可信 | 主流观点、多数命例验证 |
| 50-69 | 中等可信 | 有争议但主流认可 |
| 30-49 | 较低可信 | 某派观点、部分命例 |
| 10-29 | 参考性 | 杂格、特殊情况 |
| 0-9 | 极低/兜底 | fallback、默认值 |

### 5.2 权重设定原则

1. **有明确典籍依据 → 权重高**
2. **大量命例验证 → 权重高**
3. **主流流派共识 → 权重高**
4. **单一流派观点 → 权重中**
5. **存疑/待验证 → 权重低**
6. **兜底/默认 → 权重极低**

## 六、冲突解决规范

### 6.1 冲突类型

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 不同优先级 | 高优先级和低优先级结论不同 | 高优先级胜出 |
| 同优先级 | 同一优先级规则结论不同 | 记录冲突，取权重高的 |
| 同优先级同权重 | 完全对等的冲突 | 全部记录，交由上层判断 |

### 6.2 冲突检测

规则引擎自动检测：

```typescript
// 同优先级且结果不同 → 视为冲突
if (ruleA.priority === ruleB.priority &&
    ruleA.result !== ruleB.result) {
  addConflict(ruleA, ruleB)
}
```

### 6.3 冲突记录

所有冲突必须记录在结果中：

```typescript
interface ConflictRecord {
  rule1Id: string
  rule1Name: string
  rule2Id: string
  rule2Name: string
  resolution: 'higher_priority' | 'higher_weight' | 'both_recorded'
  chosenRuleId?: string
  reason: string
}
```

### 6.4 冲突解决流程

```
发现冲突
  │
  ├─ 优先级不同 → 高优先级胜出
  │
  └─ 优先级相同
        │
        ├─ 权重不同 → 权重高的胜出
        │
        └─ 权重相同 → 全部记录，标记为待人工审核
```

## 七、Rule 生命周期

### 7.1 生命周期阶段

```
提案 → 评审 → 开发 → 测试 → 发布 → 使用 → 优化 → 弃用
  │       │       │       │       │       │       │
  ▼       ▼       ▼       ▼       ▼       ▼       ▼
 Draft   Review    Dev    Test   Released Active  Deprecated
```

### 7.2 各阶段说明

| 阶段 | 状态标记 | 说明 |
|------|---------|------|
| 提案 | `draft` | 提出新规则，尚未评审 |
| 评审 | `review` | 正在进行规则评审 |
| 开发中 | `dev` | 正在编写代码和测试 |
| 测试中 | `test` | 正在验证准确性 |
| 已发布 | `active` | 正式生效 |
| 优化中 | `optimizing` | 调整权重/条件 |
| 弃用 | `deprecated` | 不再使用，但保留记录 |

## 八、Rule 修改流程

### 8.1 新增 Rule 流程

```
1. 提交提案
   - 规则描述
   - 典籍来源
   - 典型命例
   - 预期优先级/权重
        │
        ▼
2. 规则评审
   - 典籍考据
   - 逻辑验证
   - 冲突检查
   - 权重讨论
        │
        ▼
3. 编码实现
   - 符合 Rule 结构规范
   - 单元测试
   - 集成测试
        │
        ▼
4. 验证测试
   - 10万组随机测试
   - 典籍案例验证
   - 准确率评估
        │
        ▼
5. 发布上线
   - 更新 Rule Specification
   - 更新 Algorithm Whitepaper
   - 更新 Project Status
```

### 8.2 修改 Rule 流程

1. 提交修改申请
2. 说明修改原因（问题描述、命例证据）
3. 影响评估（影响多少命例、准确率变化）
4. 评审通过
5. 修改代码和测试
6. 全量回归测试
7. 更新版本号
8. 更新文档

### 8.3 弃用 Rule 流程

1. 发现规则有问题/不再适用
2. 标记为 `deprecated`
3. 保留至少一个版本后删除
4. 记录弃用原因
5. 更新文档

## 九、Rule Review 规范

### 9.1 评审清单

每条新规则必须经过以下评审：

- [ ] **典籍考据**：是否有明确的典籍来源？
- [ ] **逻辑自洽**：是否与现有规则体系矛盾？
- [ ] **边界清晰**：条件是否明确？边界case是否覆盖？
- [ ] **命例验证**：是否有足够的命例支持？
- [ ] **权重合理**：权重是否与可信度匹配？
- [ ] **优先级正确**：优先级是否符合层级体系？
- [ ] **测试覆盖**：单元测试是否覆盖？
- [ ] **文档完整**：Rule Specification 是否已更新？

### 9.2 评审角色

| 角色 | 职责 |
|------|------|
| 算法工程师 | 逻辑正确性、代码质量 |
| 命理顾问 | 典籍依据、专业准确性 |
| 架构师 | 体系一致性、冲突检查 |
| 测试工程师 | 测试覆盖、准确率 |

## 十一、Rule 版本管理

### 11.1 规则版本号

```
V<主版本>.<次版本>.<修订号>

例：
V6.9.0    — 当前版本
V7.0.0    — V7.0大版本
V7.0.1    — V7.0补丁
```

### 11.2 版本兼容性

- **主版本变更**：可能不兼容，规则体系有重大调整
- **次版本变更**：新增规则、调整权重，基本兼容
- **修订号变更**：Bug修复、描述优化，完全兼容

### 11.3 版本历史

| 规则版本 | 对应产品版本 | 规则总数 | 主要变化 |
|---------|------------|---------|---------|
| V1.0 | V5.0 | ~50 | 基础规则框架 |
| V2.0 | V6.0 | ~100 | 引入规则引擎 |
| V3.0 | V6.5 | ~150 | 格局/喜用神规则化 |
| **V6.9** | **V6.9** | **191** | 当前版本 |
| V7.0 | V7.0 | 400+ | 格局深化+喜用校准+旺衰合化 |
