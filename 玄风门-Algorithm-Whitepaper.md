## 第一章 算法设计理念

### 1.1 为什么不用纯AI

传统AI命理产品的问题：

| 问题 | 说明 |
|------|------|
| **不可信** | AI胡说八道，编造命理术语 |
| **不可追溯** | 黑箱输出，不知道为什么得出这个结论 |
| **不一致** | 同一张命盘，问两次结果可能不一样 |
| **不专业** | 缺少典籍依据，娱乐性质大于专业性质 |
| **无权威** | 无法验证准确性，没有可信度 |

### 1.2 三层架构设计

玄风门采用**规则引擎 + 算法推导 + AI解释**三层架构：

```
┌──────────────────────────────────────────────────┐
│  第三层：AI解释层（AI Interpreter）              │
│  - 读取算法结果，生成自然语言报告                 │
│  - 润色表达，让报告更易懂                         │
│  - 不参与任何专业判断                             │
└───────────────────────┬──────────────────────────┘
                        │ 读取 AnalysisResult
                        ▼
┌──────────────────────────────────────────────────┐
│  第二层：分析聚合层（Analysis Layer）            │
│  - 聚合各模块规则结果                             │
│  - 冲突解决与权重合并                             │
│  - 生成置信度评分                                 │
│  - 输出统一 AnalysisResult                        │
└───────────────────────┬──────────────────────────┘
                        │ 规则执行结果
                        ▼
┌──────────────────────────────────────────────────┐
│  第一层：规则引擎层（Rule Engine Layer）        │
│  - 所有命理规则统一管理                           │
│  - 可追溯：每条规则有典籍来源                     │
│  - 可验证：每条规则有单元测试                     │
│  - 可扩展：新增规则不需要改框架                   │
└──────────────────────────────────────────────────┘
```

### 1.3 核心设计原则

1. **算法第一，AI第二**
   - 算法是核心竞争力，AI只是翻译官
   - 所有专业结论来自算法，AI只负责解释

2. **每条规则可追溯**
   - 每个Rule必须有来源：哪本书、哪一派、哪一页
   - 用户可以查看：这条结论来自哪条规则
   - 专家可以审计：规则是否符合传统命理

3. **结果可验证**
   - 100000组随机测试覆盖1900-2100年
   - 与典籍案例对比验证
   - 与行业软件横向对比

4. **完全透明**
   - 推导过程全部公开
   - 没有黑箱操作
   - 置信度评分告诉用户：这个结论有多可靠

### 2.3 规则引擎核心

```typescript
// 规则统一结构
interface BaseRule<TContext, TResult> {
  id: string              // 唯一ID
  name: string            // 规则名称
  category: string        // 分类
  priority: number        // 优先级（0-999）
  weight: number          // 权重（0-100）
  description: string     // 规则描述
  reference: string       // 典籍来源
  condition: (ctx) => boolean   // 条件
  result: TResult         // 结果
}
```

**执行流程：**

```
输入 Context
    │
    ▼
按优先级排序所有规则
    │
    ▼
逐条执行 condition()
    │
    ├─ 命中 → 加入 matches
    └─ 未命中 → 跳过
    │
    ▼
检测同优先级冲突
    │
    ▼
计算最佳匹配（优先级×10 + 权重）
    │
    ▼
输出：bestMatch + allMatches + conflicts
```

## 第三章 八字排盘算法

### 3.1 排盘依据

| 项目 | 算法 | 来源 |
|------|------|------|
| **年柱** | 立春分年 + 干支纪年 | 传统命理标准 |
| **月柱** | 节气分月 + 五虎遁 | 《渊海子平》 |
| **日柱** | JDN儒略日计算 | 天文历法标准 |
| **时柱** | 时辰划分 + 五鼠遁 | 《渊海子平》 |
| **节气** | 寿星天文历（qimendunjia-standalone） | Swiss Ephemeris 精度 |

### 3.2 年柱算法

**规则：立春分年**

- 立春时刻前：上一年干支
- 立春时刻后：本年干支

**公式：**
```
年干序号 = (年 - 4) % 10
年支序号 = (年 - 4) % 12
```

**天干：** 甲乙丙丁戊己庚辛壬癸（0-9）
**地支：** 子丑寅卯辰巳午未申酉戌亥（0-11）

**参考：** 传统命理以立春为岁首，非正月初一。

### 3.3 月柱算法

**规则：节气分月**

每月以节为起点：
- 寅月：立春 → 惊蛰前
- 卯月：惊蛰 → 清明前
- 辰月：清明 → 立夏前
- 巳月：立夏 → 芒种前
- 午月：芒种 → 小暑前
- 未月：小暑 → 立秋前
- 申月：立秋 → 白露前
- 酉月：白露 → 寒露前
- 戌月：寒露 → 立冬前
- 亥月：立冬 → 大雪前
- 子月：大雪 → 小寒前
- 丑月：小寒 → 立春前

**五虎遁（月干起法）：**

| 年干 | 正月（寅）起干 | 口诀 |
|------|--------------|------|
| 甲/己 | 丙寅 | 甲己之年丙作首 |
| 乙/庚 | 戊寅 | 乙庚之岁戊为头 |
| 丙/辛 | 庚寅 | 丙辛必定寻庚起 |
| 丁/壬 | 壬寅 | 丁壬壬位顺行流 |
| 戊/癸 | 甲寅 | 戊癸何方发，甲寅之上好追求 |

**参考：** 《渊海子平·五虎遁诀》

### 3.4 日柱算法

**算法：JDN（儒略日数）计算**

```
JDN = day + floor((153m + 2)/5) + 365y
      + floor(y/4) - floor(y/100) + floor(y/400) - 32045

其中：
  a = floor((14 - month) / 12)
  y = year + 4800 - a
  m = month + 12a - 3
```

**基准：** 2000年1月1日 = JDN 2451545 = 戊午日（序号54）

**日柱序号 = ((基准序号 + (目标JDN - 基准JDN)) % 60 + 60) % 60**

**精度：** 经过1900-2100年共73000多天验证，与万年历完全一致。

### 3.5 时柱算法

**时辰划分：**

| 时辰 | 时间 | 地支 |
|------|------|------|
| 子时 | 23:00 - 01:00 | 子 |
| 丑时 | 01:00 - 03:00 | 丑 |
| 寅时 | 03:00 - 05:00 | 寅 |
| 卯时 | 05:00 - 07:00 | 卯 |
| 辰时 | 07:00 - 09:00 | 辰 |
| 巳时 | 09:00 - 11:00 | 巳 |
| 午时 | 11:00 - 13:00 | 午 |
| 未时 | 13:00 - 15:00 | 未 |
| 申时 | 15:00 - 17:00 | 申 |
| 酉时 | 17:00 - 19:00 | 酉 |
| 戌时 | 19:00 - 21:00 | 戌 |
| 亥时 | 21:00 - 23:00 | 亥 |

**五鼠遁（时干起法）：**

| 日干 | 子时起干 | 口诀 |
|------|---------|------|
| 甲/己 | 甲子 | 甲己还加甲 |
| 乙/庚 | 丙子 | 乙庚丙作初 |
| 丙/辛 | 戊子 | 丙辛从戊起 |
| 丁/壬 | 庚子 | 丁壬庚子居 |
| 戊/癸 | 壬子 | 戊癸何方发，壬子是真途 |

**参考：** 《渊海子平·五鼠遁诀》

**注意：** 子时跨日，23:00后算次日的子时（早晚子问题）。
当前实现：23:00-24:00 算当日晚子时，日柱仍用当日。

### 3.6 节气计算

**算法库：** `qimendunjia-standalone`（基于寿星天文历）

**精度：**
- 节气时刻精度：分钟级
- 覆盖范围：1900-2100年
- 与寿星天文历/Swiss Ephemeris 对齐

**缓存策略：** 按年缓存，一年计算一次，后续直接读取。

**应用场景：**
- 年柱判断（立春分年）
- 月柱判断（节气分月）
- 调候用神（节气深浅）

### 4.2 判断优先级

| 优先级 | 格局类型 | 判断条件 | 权重 |
|--------|---------|---------|------|
| 900 | 专旺格 | 一行独旺，无比劫印星以外的五行 | 95 |
| 850 | 从格 | 日主极弱，从某一行/两行 | 90 |
| 800 | 化气格 | 日干与月干/时干合化成功 | 85 |
| 700 | 外格 | 特殊格局条件 | 80 |
| 500 | 正格 | 月令取格 | 70 |
| 100 | 扶抑格 | 兜底，普通格局 | 50 |

**判断原则：**
1. 先看是否成特殊格（专旺/从格/化气/外格）
2. 特殊格不成，再看正格
3. 正格以月令为主
4. 都不满足，按扶抑格处理

### 4.3 正格判断

**取格原则：** 以月令藏干透出者为格

```
月令本气透干 → 本气为格
月令中气透干 → 中气为格
月令余气透干 → 余气为格
多干同透     → 以本气为主，兼看其他
都不透干     → 取月令本气为格
```

**各月月令藏干：**

| 月支 | 本气 | 中气 | 余气 |
|------|------|------|------|
| 寅 | 甲木（丙戊） | 丙火 | 戊土 |
| 卯 | 乙木 | — | — |
| 辰 | 戊土 | 乙木 | 癸水 |
| 巳 | 丙火 | 庚金 | 戊土 |
| 午 | 丁火 | 己土 | — |
| 未 | 己土 | 丁火 | 乙木 |
| 申 | 庚金 | 壬水 | 戊土 |
| 酉 | 辛金 | — | — |
| 戌 | 戊土 | 辛金 | 丁火 |
| 亥 | 壬水 | 甲木 | — |
| 子 | 癸水 | — | — |
| 丑 | 己土 | 辛金 | 癸水 |

**参考：** 《渊海子平·论格局》

### 4.4 从格判断

**从格条件：**
1. 日主极弱（strengthScore < 20）
2. 无比劫印星帮身（或只有一个藏干中微根）
3. 局中某一行（或两行）独旺
4. 没有破格因素

**从格分类：**

| 从格 | 条件 |
|------|------|
| 从官杀 | 官杀旺极，日主从杀 |
| 从财格 | 财星旺极，日主从财 |
| 从儿格 | 食伤旺极，日主从儿 |
| 从印格 | 印星旺极，日主从印 |

**假从格：**
- 有微弱的比劫/印星，但不足以帮身
- 从势不纯，格局层次降低

**参考：** 《滴天髓·从神章》

### 4.5 专旺格判断

**专旺格条件：**
1. 日主得令（月令为比劫印星）
2. 日主五行占绝对优势（>60%）
3. 异党极弱，不足以克制
4. 顺其气势为美，不可逆

**五专旺格：**

| 格局 | 五行 | 条件 |
|------|------|------|
| 曲直格 | 木 | 甲乙日生寅卯辰月，木旺极 |
| 炎上格 | 火 | 丙丁日生巳午未月，火旺极 |
| 稼穑格 | 土 | 戊己日生辰戌丑未月，土旺极 |
| 从革格 | 金 | 庚辛日生申酉戌月，金旺极 |
| 润下格 | 水 | 壬癸日生亥子丑月，水旺极 |

**参考：** 《三命通会·论专旺》

### 4.6 破格

**常见破格因素：**

| 格局 | 破格因素 | 影响 |
|------|---------|------|

## 第五章 喜用神算法

### 5.1 多因素模型

喜用神不是单一因素决定的，而是多因素综合加权：

```
喜用神 = f(旺衰, 格局, 调候, 病药, 通关, 扶抑, 寒暖燥湿)
```

### 5.2 各因素权重（V6.9版本）

| 因素 | 权重 | 说明 |
|------|------|------|
| **旺衰（扶抑）** | 35% | 身强克泄耗，身弱生扶比 |
| **格局** | 30% | 格局喜用，如从财喜财 |
| **调候** | 15% | 寒暖燥湿调节 |
| **病药** | 10% | 有病则用药 |
| **通关** | 5% | 两行相战，通关为用 |
| **其他** | 5% | 特殊情况 |

> ⚠️ 注意：当前权重为经验值，V7.0需要基于大量命例重新校准。

### 5.3 扶抑法

**原则：**
- 身强（旺）：喜克、泄、耗 → 官杀、食伤、财星
- 身弱（衰）：喜生、扶 → 印星、比劫

**身强身弱判断：**
- strengthScore > 60：身强
- 40 ≤ strengthScore ≤ 60：中和
- strengthScore < 40：身弱

### 5.4 格局法

**正格喜用：**
